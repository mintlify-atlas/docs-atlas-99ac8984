## Prerequisites

Before you start, make sure you have the following installed:

- **Node.js 18 or later** — [nodejs.org](https://nodejs.org)
- **npm**, **yarn**, or **pnpm** — npm ships with Node.js

## Installation

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/your-org/low-poly-ocean.git
    cd low-poly-ocean
    ```
  </Step>

  <Step title="Install dependencies">
    <CodeGroup>

    ```bash npm
    npm install
    ```

    ```bash yarn
    yarn install
    ```

    ```bash pnpm
    pnpm install
    ```

    </CodeGroup>
  </Step>

  <Step title="Start the development server">
    <CodeGroup>

    ```bash npm
    npm run dev
    ```

    ```bash yarn
    yarn dev
    ```

    ```bash pnpm
    pnpm dev
    ```

    </CodeGroup>

    Vite starts the dev server at **http://localhost:5173**. Open that URL in your browser — you will land on the Stages page.
  </Step>
</Steps>

## Available scripts

| Script | Command | Description |
| --- | --- | --- |
| `dev` | `npm run dev` | Starts the Vite dev server with hot module replacement |
| `build` | `npm run build` | Compiles and bundles the app into `dist/` |
| `preview` | `npm run preview` | Serves the production build locally for verification |
| `lint` | `npm run lint` | Runs ESLint across the source files |

## What you will see

The experience is structured as three sequential stages. When you open the app you land on `/stages`, which shows the stage selector.

<CardGroup cols={3}>
  <Card title="Stage 1 — Introduction" icon="play">
    A short animated intro sequence that sets the scene before you enter the ocean experience.
  </Card>
  <Card title="Stage 2 — Low-poly ocean" icon="fish">
    The full scrolling ocean at `/landing`, where all 10 endangered Thai marine animals appear as low-poly polygon art as you scroll.
  </Card>
  <Card title="Stage 3 — Animal detail" icon="shapes">
    Select any animal to view bilingual educational content — introduction, threats, and conservation solutions — alongside an animated polygon shard reveal.
  </Card>
</CardGroup>

<Note>
  The app redirects the root path `/` to `/stages` automatically, so you will always start at the stage selector.
</Note>

## Deploying to Vercel

The repository includes a `vercel.json` file in the project root that configures Vercel for SPA routing. All unmatched paths are rewritten to `index.html` so client-side navigation works correctly after a hard reload.

<Steps>
  <Step title="Install the Vercel CLI (optional)">
    ```bash
    npm i -g vercel
    ```
  </Step>

  <Step title="Deploy">
    Run the following command from the project root, or connect your Git repository through the Vercel dashboard:

    ```bash
    vercel
    ```

    Vercel automatically detects the Vite framework, runs `npm run build`, and serves the `dist/` directory.
  </Step>
</Steps>

<Tip>
  You can also deploy by importing the repository directly in the [Vercel dashboard](https://vercel.com/new). No extra configuration is needed — `vercel.json` handles everything.
</Tip>
